Documentación Técnica

Guía de arquitectura y funcionamiento interno de EmpresaScout.

1. Arquitectura del Sistema (Cliente-Servidor)

EmpresaScout utiliza un modelo cliente-servidor asíncrono para garantizar un rendimiento fluido durante procesos de larga duración.

  • Frontend: Desarrollado en HTML5, CSS3 (Glassmorphism) y Vanilla JavaScript. Se encarga de la interfaz y visualización de datos.
  • Backend: Desarrollado en Python con el micro-framework Flask. Gestiona el ciclo de vida de los agentes y las sesiones.
  • Concurrencia: Los procesos pesados (Scraping y Enriquecimiento) se ejecutan en Threads (hilos) en segundo plano para no bloquear el servidor principal.
  • Comunicación: El cliente y el servidor se comunican en tiempo real mediante Server-Sent Events (SSE), lo que permite enviar filas de datos al navegador conforme se van encontrando, sin tener que esperar a que el proceso termine por completo.

2. Flujo de Scraping (ScraperAgent)

El motor de búsqueda ha sido optimizado para la máxima velocidad y precisión, buscando empresas directamente sin pasar por intermediarios.

  1. Búsqueda Directa: Utiliza la librería duckduckgo_search (DDGS) con comandos de búsqueda avanzada ("prompts") para localizar directamente las páginas web corporativas de empresas del sector elegido.
  2. Procesamiento Paralelo (Multithreading): Para lograr una velocidad de extracción masiva, se emplea un ThreadPoolExecutor. El agente abre, lee y analiza de forma segura y concurrente múltiples páginas web al mismo tiempo (controlado por un threading.RLock() para evitar condiciones de carrera).
  3. Extracción y Validación Inteligente: Se implementa un filtro heurístico mediante palabras clave. El sistema analiza el texto del DOM de cada página; si no se detectan términos semánticos vinculados al sector seleccionado, la web es descartada como falso positivo.
  4. Reconocimiento de Patrones: Si la web pertenece al sector, se aplican Expresiones Regulares (RegEx) para barrer el HTML e identificar heurísticamente datos de contacto como emails y números de teléfono.

3. Flujo de Enriquecimiento (EnricherAgent & AI)

El agente de enriquecimiento permite completar bases de datos parcialmente vacías usando Inteligencia Artificial.

  • Lectura de Datos: Se utiliza Pandas para soportar múltiples formatos (.csv, .xlsx) detectando automáticamente las columnas más probables ("Empresa", "Web").
  • Interacción con el Navegador: Si el modo de IA está desactivado, el sistema usará un parser estático.
  • Motor de Inferencia (Anthropic API): Cuando se provee una clave API (Claude), el sistema inyecta HTML simplificado y metadatos de las páginas de la empresa directamente en un LLM de Anthropic mediante un prompt especializado que devuelve exclusivamente estructuras JSON. Esto maximiza la precisión en la recuperación de emails corporativos.
